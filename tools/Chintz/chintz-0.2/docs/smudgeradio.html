Smudge: The RadioButton Tag

RadioButtons provide a means, from within an enclosing Buttons container, for setting child switches which can take integer values.One RadioButton out of a group can be set Checked initially ,as for CheckBoxes. Values are assigned to the variable defined by the enclosing < Buttons> container in order, with the first button listed setting the variable to 0, the second setting to 1 and so on. RadioButtons have a Geometry relative to their container, and a Title which can be Bold, and can set Switches in the same way as CheckBoxes.
Example:-

< Page>
< Title> Setup < /Title>
    < Frame,id=Firstframe>
      < Geometry> 10 10 100 200 < /Geometry>
      < Enabled> false < /Enabled>
    < /Frame>

    < Buttons,id=OPT_PT_PLACE>
      < Style> NoFrame < /Style>
      < Geometry> 10 210 100 200 < /Geometry>
      < Enabled> false < /Enabled>
      < Title> Point Placement < /Title>

      < RadioButton>
      < Geometry> 10 10 70 20 < /Geometry>
      < Title> Uniform < /Title>
      < Checked> true < /Checked>
      < Switch> Firstframe false < /Switch>
      < /RadioButton>

    < /Buttons>
< /Page>

a page, labelled "Setup", containing a frame, and a Button container labelled "Point Placement", which in turn contains a single RadioButton, labelled "Uniform". The Button container allows the setting of OPT_PT_PLACE. The RadioButton is initially selected, and controls the enabling of Firstframe via the < Switch> tag. When the RadioButton is Checked, the Frame will be disabled. Again, it is advisable to define Switches after the id of the Frame or Box is defined (i.e. put the Switch further down the smudge.ui file than the Frame) to avoid possible problems with undefined pointers when chintz is run.
