Smudge: The BiLine Tag

The BiLine tag is the third way to set up child control variables (the remaining one is < CheckBox>). In this case, the only additional tag available,which applies only to the BiLine is < Origin> Xorigin Yorigin < /Origin>, which defines the BiLine origin relative to its container. There is no id for a BiLine. A BiLine provides two text boxes, the first labelled "Variable" and the second "Value". The overall size is 150 wide and 80 deep. When entries are made into both boxes, then saving from chintz writes

Variable text:
Value text

to the child control file, where "Variable text" represents what was typed into the Variable text box, and "Value text" that typed in the value box. This provides the user of chintz with a way of defining new variables that are not otherwise catered for in the chintz UI.

Example:-

< Page>
< Title> Setup < /Title>
    < Frame,id=Firstframe>
      < Style> NoFrame < /Style>
      < Geometry> 10 10 100 200 < /Geometry>
      < Enabled> false < /Enabled>
      < Title> Mesh < /Title>

      < BiLine>
        < Origin> 80 10 < /Origin>
      < /BiLine>

    < /Frame>
< /Page>

a page, labelled "Setup", containing a frame labelled "Mesh", which in turn contains a BiLine, starting 80 pixels in X and 10 in Y into the Frame.
