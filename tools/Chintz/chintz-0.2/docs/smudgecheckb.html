Smudge: The CheckBox Tag

CheckBoxes provide a means of writing variables to the child control file which may take on values 0 or 1. As with Buttons or Lines, the variable name is specified via the id. CheckBoxes have a Geometry relative to their container, and a Title which can be Bold. There is one futher tag which can be used with CheckBoxes and RadioButtons, which is
< Switch> Identifier true/false < /Switch>
Here the Identifier refers to a previously labelled Frame or Box. If the value true is chosen, then the Frame or Box will be enabled when the CheckBox is checked, otherwise it will be disabled. Using the value false will disable the Frame or Box when the CheckBox is Checked. The initial state of the CheckBox can be determined using the Checked tag.
Example:-

< Page>
< Title> Setup < /Title>
    < Frame,id=Firstframe>
      < Geometry> 10 10 100 200 < /Geometry>
      < Enabled> false < /Enabled>
    < /Frame>

    < Frame,id=Anotherframe>
      < Style> NoFrame < /Style>
      < Geometry> 10 210 100 200 < /Geometry>
      < Enabled> false < /Enabled>
      < Title> Mesh < /Title>

      < CheckBox,id=LAKEFILL>
      < Geometry> 10 10 70 20 < /Geometry>
      < Title> Fill Lakes < /Title>
      < Checked> true < /Checked>
      < Switch> Firstframe true < /Switch>
      < /CheckBox>

    < /Frame>
< /Page>

a page, labelled "Setup", containing a two frames, one labelled "Mesh", which in turn contains a CheckBox labelled "Fill Lakes", allowing the setting of LAKEFILL to 0 or 1. The Checkbox is initially checked, and controls the enabling of Firstframe via the < Switch> tag. It is advisable to define Switches after the id of the Frame or Box is defined (i.e. put the Switch further down the smudge.ui file than the Frame) to avoid possible problems with undefined pointers when chintz is run.
