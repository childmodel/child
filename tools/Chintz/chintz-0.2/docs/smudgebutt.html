Smudge: The Buttons Tag

Buttons provide a special container within which the user can place clickable RadioButtons. The RadioButtons are exclusive, that is only one of them can be selected at any time. As with the previous containers, the Buttons container can have a Title, which may be Bold, and a Geometry. However, the Buttons container cannot be switched :- to do this, it must be embedded within another switchable element, such as a Frame. This is to allow a secondary use for the id, which is to identify a variable whose value is to be written to the child control file.
Example:-

< Page>
< Title> Setup < /Title>
    < Frame,id=Firstframe>
      < Style> NoFrame < /Style>
      < Geometry> 10 10 100 200 < /Geometry>
      < Enabled> false < /Enabled>
      < Title> Mesh < /Title>

      < Buttons,id=OPT_PT_PLACE>
        < Geometry> 10 10 70 100 < /Geometry>
        < Title> Open Boundary Type < /Title>

        < RadioButton>
            < Geometry> 10 10 70 20 < /Geometry>
            < Title> Corner < /Title>
        < /RadioButton>

        < RadioButton>
            < Geometry> 10 30 70 20 < /Geometry>
            < Title> 1 Side < /Title>
        < /RadioButton>

      < /Buttons>

    < /Frame>
< /Page>

a page, labelled "Setup", containing a frame labelled "Mesh", which in turn contains a Buttons container labelled "Open Boundary Type". This title will be inset into the Buttons border, as for the Box container. The Buttons geometry is again relative to the Frame which contains it, so its origin is 10 pixels into the frame. There are two RadioButtons in the box, with labels "Corner" and "1 Side". The RadioButton Geometry sets their positions in the Box. In this case the RadioButtons are aligned in X, but are placed one above another using the Yorigin. When the "Corner" RadioButton is selected, the value 0 will be assigned to the variable OPT_PT_PLACE, as defined by the id of the Buttons container. When the control file is saved, the text

OPT_PT_PLACE:
0

will be written to it. Similarly, selecting the second button will give OPT_PT_PLACE value 1. In this way the options for child can be set up as required. Note that if the id is omitted, then an internally generated variable name (e.g. Buttons1var) will be written to the control file.
